CSS——层叠

2018-05-18

层叠顺序

relative(只要定位了，都是要按index计算) -> 文字/内联元素/inline-block -> float(包裹float内的内联元素与float同一层叠级别) -> div/块级元素 -> 【position(z-index=-1)(父元素定位，且设置z-index)】 ->border -> backgroud -> 【position(z-index=-1)(父元素未定位)】

  • 同级别的，后出现的会盖住先出现的
  • 只有position不为static的元素，设置z-index才有用。

    堆叠上下文

  1. 以下这些元素(不全)会触发堆叠上下文：
  • 根元素(html)
  • z-index不为auto(默认为auto)的绝对/相对定位元素
  • z-index不为auto(默认为auto)的flex元素
  • position:fixed
  • opacity属性小于1的元素
  1. 作用：主要影响z-index
  • 没有其他堆叠上下文的情况下，html会管理堆叠上下文的关系。
  • 满足任一一个都会触发一个新的堆叠上下文，内部子元素与外部堆叠上下文的元素比较时，其层级由父元素(管理自己的堆叠上下文)决定，内部子元素之间互相堆叠顺序不变。
    • 这也解释了为什么【position(z-index=-1)(父元素定位，且设置z-index)】的层级顺序高于background和border。
