AJAX基础

2018-04-23

  • 发展史
  1. form可以发送GET和POST请求，会刷新页面或新开页面
  2. a可以发送GET请求，会刷新页面或新开页面
  3. img可以发送GET请求，只能以图片形式展示
  4. link可以发送GET请求，只能以css或favicon形式展示
  5. script可以发送GET请求，只能以脚本形式运行
  6. IE5在JS中引入ActiveX对象(API)，使JS可以直接发送请求，其他浏览器跟进，取名XMLHttpRequest，被纳入W3C规范。现在进化成AJAX
  • AJAX：异步的JavaScript和XML。
  1. 使用XMLHttpRequest发请求
  2. 服务器返回XML格式的字符串，只能返回字符串
  3. JS解析XML，并更新局部页面

API

send()

send方法用于实际发出HTTP请求。如果不带参数，就表示HTTP请求只包含头信息，也就是只有一个URL，典型例子就是GET请求；如果带有参数，就表示除了头信息，还带有包含具体数据的信息体，典型例子就是POST请求。

setRequestHeader

readyState

监听readyState来判断请求的状态,通过onreadystatuschange事件判断readyState的值

同源策略

form、img、script等请求方式，是可以对别的域名发送请求的。
但用AJAX不能对别的域名放送请求，只有协议、域名、端口号一摸一样才允许发送AJAX（多个www是不同的域名）
因为AJAX是可以获取响应内容的，如果没有同源策略的限制，别的网站可以通过AJAX获取另一个网站的所有信息。

CORS

跨域资源共享：Cross-Origin Resource Sharing

  • AJAX可以通过CORS发送请求到别的网站，基本思想是使用自定义的HTTP头部让浏览器与服务器进行沟通，从而决定请求或响应式应该成功还是失败

JSONP

可以让网页从别的域名（网站）那获取资料，即跨域读取数据。可以理解为被包含在函数调用中的JSON

为什么JSONP不支持post请求

  • 跨域引入js脚本文件是可以的，JSONP是通过动态创建script的方法进行的，然后利用src属性进行跨域，script只能发送get请求不能发送post请求。

JSONP与CORS

  • JSONP只能实现GET请求，而CORS支持所有类型的HTTP请求
  • 使用CORS，开发者可以使用普通的XMLHttpRequest发起请求和获得数据，比起JSONP有更好的错误处理
  • JSONP主要被老的浏览器支持，它们往往不支持CORS，而绝大多数现代浏览器都已经支持了CORS

安全问题

CSRF与XSS

  • 攻击手段类似，客户端执行而已代码
  • CSRF不注重与获取用户Cookie
  • CSRF不仅可以在源站发起攻击，还可以引导用户方位其他危险网站的同时发起攻击

    CSRF

  • CSRF攻击是源于WEB的隐式身份验证机制，WEB的身份验证机制虽然可以保证一个请求是来自于某个用户的浏览器，但却无法保证该请求是用户批准发送的。
  • 对于CSRF的防御也分为服务端防御与客户端防御两种，服务端防御典型的譬如给某个页面添加随机数，使得无法从第三方页面直接提交。
  • 在客户端防御的话可以利用譬如Firefox提供的一些检查工具。注意，CSRF并没有打破同源策略。

    XSS

  • XSS全程是跨站脚本攻击，即攻击者向某个Web页面中插入恶意的JavaScript脚本，而当普通用户访问时，该恶意脚本自动执行而从盗取用户的Cookie等信息。
  • 对于XSS的防御手段主要就是输入检查与输出检查。
  • 譬如对用户输入的文本框内容进行<、>这样的特殊字符检查。
  • 输出检查则是指对于输出到网页的内容进行过滤或者编解码，譬如使用HTML编码将<转义。
