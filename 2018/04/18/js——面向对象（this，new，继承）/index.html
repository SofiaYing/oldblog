JS面向对象

面向对象的三大特点：

  • 封装：方便合作，减轻负担。【大部分语言用类实现；js中没有类，用的是构造函数（返回为一个对象的函数）。】
  • 继承：可以复用代码。【js中ES6之前用原型链实现共有属性继承】
  • 多态：让代码更灵活。

对象：某个特定引用类型的实例。

  • 普通对象
  • 数组：下标为‘0’到‘length’的有序字符串
  • 函数

原型：理解为公有属性

  • 注意：访问proto会造成性能上的降低。
  • 函数和对象是没有关联的，一般来说this就表示调用它的这个对象。
  • 参数的值只有在传参的时候才能确定，this是第一个参数 => this的值只有的传参的时候才能确定

this

1.默认绑定

  • 最常用的函数调用类型：独立函数调用

2.隐式绑定

  • 当函数引用有上下文对象时，隐式绑定规则会把函数调用中的this绑定到这个上下文对象。

  • 对象属性引用链中只有上一层或者说最后一层在调用位置中起作用。

  • 隐式丢失问题：即被隐式绑定的函数会丢失绑定对象，就会应用默认规则

new

new帮忙做三件事

(1)js已经帮忙初始化一个空对象this，不需要自己去想变量名

(2)帮忙创建好obj. proto = creatSoldier.prototype, 同时会为其自动创建constructor属性，表述了所有的实例都是由creatSoldier创建的。

(3)已经帮忙返回this，不需要自己去写。如果自己写了return,如return {}，js会帮忙

  • 约定俗成的一些事：
  1. 构造函数首字母大写 CreatSoldier
  2. 构造函数省略creat 因为new已经代表了creat的意思 Solider
  3. 如果构造函数没有参数，可以省略括号 sodliers.push(new Solider)
