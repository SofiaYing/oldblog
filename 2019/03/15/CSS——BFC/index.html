CSS——BFC

2019-03-15

  • 块级格式化上下文（block formatting context）
  • 块格式化上下文对于定位 (参见 float) 与清除浮动 (参见 clear) 很重要。浮动定位和清除浮动时只会应用于同一BFC的元素。外边距折叠也只会发生在属于同一BFC的块级元素之间。
  • 同一个块格式化上下文中的相邻块级盒之间的竖直margin会合并。（margin合并特性，跟BFC没什么关系）
  • 一个块格式化上下文包括创建它的元素内部所有内容，除了被包含于创建新的块级格式化上下文的后代元素内的元素。(即BFC爸爸管儿子，儿子如果也是BFC，爸爸不管孙子)
  • 任何情况下都可以不使用BFC（清除浮动、左右自适应布局、margin穿透父元素）

1.触发BFC条件

  • 根元素或者包含它的元素
  • float（除了none）
  • overflow（除了visible）
  • display（table-cell/table-caption/inline-block）(即不是block的block)
  • position：absolute
  • display: flow-root (可以创建无副作用的BFC)

2.BFC功能

  • BFC包裹内部元素：浮动元素（带来了清除浮动的效果，并不是真的清楚浮动，为了清除浮动可能带来副作用）

  • 相邻BFC界限分明：用 float + div 做左右自适应布局（其实flex就能做）

  • BFC包裹内部元素：父子margin合并问题(当父元素垂直方向没有能挡住子元素的margin时，子元素margin和父元素的margin合并，加上border之后)

    注意： 虽然将父元素设置为BFC也能解决这一问题（BFC会包裹内部元素），但是会带来副作用，不推荐。
